# 👁️ Real-Time Drowsiness / Eye Closure Detector

Detects sustained eye closure using your webcam and fires an alarm when eyes
stay closed beyond a configurable threshold (default **2 seconds**).

Built with **OpenCV** + **MediaPipe Face Mesh** + **Eye Aspect Ratio (EAR)**.

## ⚙️ Configuration

Open **`config.py`** and tweak:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `CAMERA_INDEX` | `0` | Which camera to use (`0` = built-in, `1` = external) |
| `FRAME_WIDTH/HEIGHT` | `640×480` | Capture resolution |
| `EAR_THRESHOLD` | `0.22` | EAR below this = eyes closed (raise if false positives) |
| `ALARM_THRESHOLD_SEC` | `2.0` | Seconds of closure before alarm triggers |
| `ALARM_SOUND_PATH` | `sounds/alarm.wav` | Path to alarm audio file |

## 🔊 Alarm Sound Details

The alarm module tries these backends in order:

1. **pygame** — best quality, loops smoothly
2. **playsound** — simple WAV/MP3 playback
3. **subprocess** (`paplay` / `aplay` / `mpg123`) — Linux native
4. **sounddevice** — generates a sine-wave beep, no WAV file needed
5. **Terminal bell** — silent fallback

If you have PulseAudio/PipeWire issues:

```bash
# Check audio is working
paplay /usr/share/sounds/freedesktop/stereo/bell.oga

# Re-initialise PulseAudio
pulseaudio --kill && pulseaudio --start
```

## 🐛 Troubleshooting

### Camera not found
```
RuntimeError: ❌ Cannot open camera index 0
```
- Try `CAMERA_INDEX = 1` or `2` in `config.py`
- Check: `ls /dev/video*`
- Ensure no other app is using the camera (`fuser /dev/video0`)

### `libGL.so.1: cannot open shared object file`
```bash
sudo dnf install mesa-libGL          # Fedora
# OR
sudo apt install libgl1-mesa-glx     # Ubuntu
```

### MediaPipe import error
```bash
pip install --upgrade mediapipe
```
If on ARM (Pi): `pip install mediapipe-rpi4`

### No sound / alarm silent
```bash
# Test system audio
speaker-test -t sine -f 1000 -l 1
# Check PulseAudio
pulseaudio --check -v
# Force sounddevice backend by removing pygame:
pip uninstall pygame
```

### EAR always low (eyes always "closed")
- Increase lighting — MediaPipe struggles in low light
- Adjust `EAR_THRESHOLD` down to `0.18` in `config.py`
- Ensure face is centred and camera is at eye level

### High CPU usage
- Lower resolution: `FRAME_WIDTH = 320`, `FRAME_HEIGHT = 240`
- MediaPipe already uses hardware acceleration when available

## 🧠 How it Works

```
Camera frame
    │
    ▼
MediaPipe Face Mesh  (468 landmarks, 1 ms/frame on CPU)
    │
    ▼
Extract 6 eye landmarks (per eye)
    │
    ▼
Compute EAR = (‖p2-p6‖ + ‖p3-p5‖) / (2 × ‖p1-p4‖)
    │
    ├── EAR ≥ threshold  →  eyes OPEN  →  reset timer
    │
    └── EAR < threshold  →  eyes CLOSED
            │
            └── duration > ALARM_THRESHOLD_SEC  →  🚨 ALARM
```

EAR formula: *Soukupová & Čech (2016)*
